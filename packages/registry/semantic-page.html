Semantic CSS Classes with Tailwind @apply

Transform verbose Tailwind utility classes into clean, semantic CSS using @apply directives. This approach reduces code volume while maintaining the same design system and improving readability for developers.

The BuildY UI8Kit provides a powerful CLI for adding UI components with multi-registry support. Install components via npx buildy-ui@latest init and add semantic components with npx buildy-ui@latest add button --registry semantic. For production builds, use npm package installation to process Tailwind directives, or use our compiled CSS via CDN for quick prototyping.

Clean Code Benefits

Semantic classes like .button-primary replace dozens of utility classes, making your HTML cleaner and more maintainable. Instead of writing bg-primary text-primary-foreground shadow-xs hover:bg-primary/90, simply use button-primary.

Tailwind Directive @apply

Use @apply to extract common utility patterns into reusable semantic classes that express intent rather than implementation. This creates a design system that's both developer-friendly and maintainable across large projects.
The CLI supports utility, semantic, and theme registries with intelligent dependency validation, ensuring components work together seamlessly.
